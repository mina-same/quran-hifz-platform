- [2026-07-03] Added track-card grids in `quran-hifz/src/quran/pages/**/*SpecialTracks.tsx` using inline `gridTemplateColumns: "repeat(auto-fill,minmax(380px,1fr))"` with no `className="grid-collapse"` ‚Äî on a phone narrower than 380px this forces a track wider than the viewport (CSS Grid doesn't shrink a fixed `minmax()` floor below the container), causing horizontal overflow/scroll instead of stacking. - [2026-07-03] When a form submits a nested array of objects with a backend-required field (e.g. `QuranPlan.pointRules[].label` ‚Äî Zod `min(1)` server-side), always add a matching client-side pre-submit check. Skipping this on the teacher plan-builder's "add point rule" button let a user add an empty-label row, click submit, and get nothing but the generic `"ÿ®ŸäÿßŸÜÿßÿ™ ÿ∫Ÿäÿ± ÿµÿßŸÑÿ≠ÿ©"` message with the modal just silently staying open ‚Äî cost a full extra E2E debug cycle to trace via network-response logging back to the missing `label`.
